There are two hash tables inside the JDBM. 1. ht12. ht2ht1: key = word in documents    content returned = arraylist<(Content) Object>  (Object class contain 2 members :  1st. URL, 2nd. Arraylist<int POSITION inside URL> )ht2:key = URL    content returned=(Doc) Object (Doc class contain 5 members : 1st doc number, 2nd title, 3rd last modified date, 4th size, 5th arraylist<string childlink>)ht1 for storing words.Ht2 for storing website details, reason for using url as key because URL is unique for each websites.
